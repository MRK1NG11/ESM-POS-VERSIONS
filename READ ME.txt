=============================================================================
 CLOUD INFRASTRUCTURE SETUP (CRITICAL)
=============================================================================
    To enable multi-till syncing, the system uses Google Drive as a "Live Bridge."

STEP-BY-STEP INSTALLATION:

    INSTALL GOOGLE DRIVE: Install the "Google Drive for Desktop" app on the
    Backoffice PC and all Till PCs.

    CREATE DATA HUB: On the Backoffice PC, create a folder named
    "ESM_CLOUD_DATA" inside your Google Drive.

    CONFIGURE TILLS: On each terminal, open "local_config.json" and set:

        TILL_NUMBER: "01" (Set unique IDs for each terminal).

        NETWORK_DATA_FOLDER: "G:/My Drive/ESM_CLOUD_DATA"
        (Ensure the drive letter matches your Google Drive setup).

3. CORE TILL OPERATIONS
=============================================================================
    PROCESSING A SALE:
    Scan a barcode. The "Smart Verify Strip" instantly displays Name, Stock,
    and Price. The input field auto-clears for maximum scanning speed.

    CLOUD MANUAL SELECT (MS):
    Browse the visual catalog. Categories (MEAT, SNACKS, etc.) and prices are
    pulled live from the cloud. If you change a price in the office, it
    reflects here immediately.

    SMART TENDER SYSTEM:

        8 QUICK-CASH BUTTONS: Rapidly input cash from R200 down to R1.

        LCD FEEDBACK: Screen glows RED (Cash Due) or NEON GREEN (Change Due).

    TILL PAY OUTS (2ND FUNC):
    Securely remove cash for store expenses. Every cent is logged in the
    Cloud Pay-Outs Ledger with the associated User, Time, and Reason.

5. VERSION 0.6.2: THE "NETWORK" UPDATE
=============================================================================
This release marks the transition from a standalone register to a
fully-linked store network.

MAJOR UPGRADES:

    Google Drive Data Bridge: Replaces unreliable local networking with stable,
    retry-protected cloud syncing.

    File Collision Prevention: Each till saves its own unique file
    (e.g., transactions_01.json), preventing data corruption during
    simultaneous sales.

    Global Staffing: Staff IDs and Admin PINs are now pulled from a central
    file. No more updating every PC manually when you hire a new cashier.

    Retry Logic: The code now "waits" for Google Drive to finish syncing before
    reading/writing, ensuring 100% data integrity.

    Fullscreen Kiosk Mode: Total security. Users cannot access the Windows
    desktop without an Admin exiting the app via the secure menu.
