🥩 EAT SUM MEAT - Point of Sale (POS) System

Store: Eat Sum Meat (Pty) Ltd | Current Build: v0.3.4
📌 1. System Overview

The Eat Sum Meat POS is a custom-built, offline-first terminal designed for high-speed retail checkout. It features a locked-down Kiosk Mode, automated cloud updating, hardware-level silent receipt printing, and secure End-of-Day (EOD) data syncing.

⚙️ 2. First-Time Setup & Installation

Because this software generates local databases and handles secure system-level updates, it must be installed correctly.

    Create a Dedicated Folder: Do not run the .exe from your Downloads folder. Move EatSumMeatPOS.exe to a permanent location (e.g., C:\POS_System\).

    Run as Administrator: Right-click the .exe and select "Run as Administrator". The software requires this to execute the Auto-Updater and focus the barcode scanner.

    Automatic File Generation: Upon first boot, the system will automatically generate products.json, transactions.json, and sessions.json in the same folder. Do not delete these files.


🧑‍💻 3. Operating Instructions
Logging In

    The system is locked behind a secure login screen.

    Default Cashier Codes: CSH1, CSH2, ADMIN

    Upon successful login, the barcode scanner is automatically engaged.

Ringing Up Customers

    Scan: Use a physical barcode scanner, or type the code and press ENT on the digital numpad.

    Quick Cash: Use the exact amount buttons (e.g., +R50, +R100) to quickly calculate change.

    Payment: Click PAY CASH or PAY EFT. The system will silently spool the receipt to the default Windows printer and clear the till for the next customer.

    Voiding: Select an item in the cart and click VOID. This requires an Admin override.

The Admin Panel

    Click ADMIN PANEL at the bottom right.

    Default Manager PIN: 1981

    From here, you can view live inventory, today's local sales, and cashier login durations.


☁️ 4. Cloud & Network Features

Over-The-Air (OTA) Updates
The POS will check for new versions on every boot. If a new version is detected on the company GitHub repository, the app will prompt the cashier, download the update, swap the .exe files, and reboot itself automatically.

End of Day (EOD) Sync
At the end of a shift, a manager must open the Admin Panel and click "RUN EOD (SYNC PREP)". This will securely package all local sales, sessions, and inventory data, stamp it with the Till ID and Date, and push it to the Back Office sync folder.
